# AltSwitcher: План Разработки Приложения

## 1. Резюме и Цель (Executive Summary & Goal)
**AltSwitcher** — это фоновое macOS-приложение, предназначенное для автоматического и ручного исправления текста, набранного в неправильной языковой раскладке (например, `ghbdtn` -> `привет`).
**Главная цель:** Создать надежный, быстрый и безопасный аналог Punto Switcher, который работает на современных версиях macOS (с процессорами Apple Silicon), интегрируется с нативными системными API и не вызывает задержек при печати.

## 2. Архитектура и Основные Компоненты
- **Тип приложения:** Agent / LSUIElement (работает в фоне, не отображается в Dock, имеет иконку в Menu Bar).
- **Стек:** Swift 5/6, AppKit (для ядра и Menu Bar), SwiftUI (для окна настроек).
- **Глобальный перехват ввода (Keylogger):** `CGEventTapCreate` (Quartz Event Services).
- **Маппер раскладок:** Модуль конвертации строк между QWERTY и ЙЦУКЕН (и обратно).
- **Управление системным вводом:** `TISSelectInputSource` для переключения текущего источника ввода.
- **Генератор событий:** `CGEventPost` для симуляции нажатий `Backspace` и ввода корректных символов.

## 3. Этапы Разработки (Phases / Milestones)

### Этап 1: Proof of Concept (Ядро и перехват)
*Фокус: проверка технической возможности низкоуровневого управления вводом.*
- [ ] Инициализация macOS-проекта без главного окна.
- [ ] Добавление запроса прав `Accessibility` при старте приложения.
- [ ] Написание `CGEventTap`, который просто логирует нажатия клавиш в консоль.
- [ ] Создание кольцевого буфера для сохранения последних введенных символов.

### Этап 2: MVP Функционал (Ручное переключение)
*Фокус: реализация магии переключения по горячей клавише.*
- [ ] Реализация словаря/логики конвертации символов (Eng <-> Rus).
- [ ] Добавление глобального слушателя "горячей клавиши" (например, `Pause/Break` или `Option+Space`).
- [ ] При нажатии горячей клавиши:
  - Вычисление длины последнего слова в буфере.
  - Программная отправка серии `Backspace` для стирания слова.
  - Переключение системной раскладки через API.
  - Программная отправка правильного слова.

### Этап 3: Автоматическое переключение и UI
*Фокус: эвристика и интерфейс пользователя.*
- [ ] Разработка эвристики "абракадабры": если юзер набирает невозможные для текущего языка сочетания букв, автоматически запускать процесс стирания и замены.
- [ ] Создание иконки в Menu Bar (`NSStatusItem`) с выпадающим меню.
- [ ] Создание окна настроек (Preferences) на SwiftUI:
  - Чекбокс "Автоматическое переключение".
  - Настройка "Горячая клавиша для ручного переключения".
  - Таблица "Исключения" (Список Bundle ID приложений, где AltSwitcher временно отключается — IDE, терминал, игры).

### Этап 4: Полировка и Релиз
*Фокус: безопасность, стабильность и упаковка.*
- [ ] Добавление опции автозагрузки (Launch at Login).
- [ ] Настройка безопасности: очистка буфера при неактивности или смене фокуса окна, чтобы не хранить пароли.
- [ ] Подпись кода (Code Signing) и Notarization.
- [ ] Создание установочного `.dmg` образа.

## 4. Риски и Возможные Проблемы
- **Права доступа:** macOS очень строго относится к перехвату клавиатуры (Input Monitoring, Accessibility). Необходимо сделать запрос прав понятным для юзера, иначе приложение не будет работать.
- **Производительность:** `CGEventTap` отрабатывает синхронно для каждого нажатия. Если логика заблокирует поток, клавиатура будет "лагать". Нужно, чтобы алгоритмы работали молниеносно (О(1)).
- **Пароли:** Мы не должны перехватывать ввод в полях с паролями (Secure Text Fields). Обычно macOS сама блокирует EventTap в `NSSecureTextField`, но это нужно дополнительно протестировать.

## 5. Критерии Готовности (Definition of Done)
Приложение готово к релизу, когда:
- Оно устанавливается из `.dmg` файла без блокировок Gatekeeper.
- Пользователь может успешно печатать текст, и приложение корректно стирает и заменяет слова по горячей клавише.
- Встроенный UI позволяет добавлять приложения в черный список (где переключение игнорируется).
- Нет заметных задержек при обычной печати.
